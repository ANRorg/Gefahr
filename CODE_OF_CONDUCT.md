# Code of Conduct

Gefahr is infrastructure software. Project discussion should stay technical,
direct, and respectful so operators can trust the work and contributors can
make progress.

## Expected behavior

- Be specific when reporting bugs, reviewing code, or challenging a design.
- Assume operational impact matters; explain failure modes and tradeoffs.
- Keep disagreement focused on code, docs, tests, deployment behavior, and user
  outcomes.
- Credit prior work and avoid misrepresenting project capabilities.
- Give maintainers enough context to reproduce issues.

## Unacceptable behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private information or credentials.
- Reporting security vulnerabilities in public issues before maintainers have a
  chance to assess and coordinate disclosure.
- Spam, low-effort promotion, or repeated off-topic comments.

## Enforcement

Maintainers may edit, hide, or remove comments; lock threads; close issues; or
block contributors who repeatedly violate this policy. Security-sensitive
content may be moved to a private advisory without public discussion.

Report conduct concerns privately to the repository administrators.
