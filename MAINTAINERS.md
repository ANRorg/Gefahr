# Maintainers

Gefahr maintainers are responsible for code review, release readiness, security
triage, and keeping the project scope honest.

## Current ownership

Primary repository owner:

- @AnouarMohamed

When the `ANRorg` organization has a dedicated maintainer team, update
[.github/CODEOWNERS](.github/CODEOWNERS) to reference that team.

## Review areas

| Area | Review focus |
|---|---|
| Proxy runtime | Routing, forwarding, retries, limits, timeouts, cache, policies |
| Operations | Deployment manifests, release gates, runbooks, observability |
| Security | Admin auth, TLS handling, dependency updates, vulnerability response |
| Documentation | Accuracy, support boundaries, examples, production checklist |

## Becoming a maintainer

Maintainer access should be granted only after repeated high-quality
contributions that demonstrate:

- Production-oriented judgment.
- Careful review of failure modes.
- Good test and documentation discipline.
- Respect for project scope and security process.
